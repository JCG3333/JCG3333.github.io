### 개요

AWS EC2 환경에서 더미 자동매매 봇을 지속적으로 운영하기 위한 구조를 구축하고,  
별도로 뉴스 데이터 기반 감성 분석 모델을 실행하여 데이터를 지속적으로 수집·축적하고 있습니다.

두 시스템은 독립적으로 동작하지만,  
“작은 기능을 안정적으로 돌리고, 데이터를 꾸준히 쌓아 나가는 경험”이라는 공통된 목적을 가지고 있습니다.

---

### 자동매매 봇 운영 환경

더미 자동매매 봇은 단순 이동평균을 기반으로  
BUY / SELL / HOLD 신호를 로그로 출력하는 간단한 구조를 가지고 있습니다.

운영 환경 구축 시 다음 요소를 고려했습니다:

- EC2 인스턴스 생성 및 환경 구성
- Python 기반 봇 배포
- systemd를 이용한 자동 실행 및 재시작 정책 설정
- 로그 기록 및 모니터링

이를 통해 서버 재부팅, SSH 종료 등 외부 영향에도  
서비스가 자동으로 복구되도록 구성했습니다.

> "코드를 실행하는 것"보다  
> "코드가 항상 살아있게 만드는 것"에 집중한 경험입니다.

---

### Oracle Cloud에서의 초기 구현 경험

AWS 이전에는 Oracle Cloud 무료 VPS 환경(1 vCPU, 1GB RAM)에서  
같은 구조를 작은 규모로 먼저 실험했습니다.

리소스가 매우 제한적이었기 때문에:

- 패키지 최소 설치
- 로그 순환
- 출력 간격 조정
- 메모리 사용 최적화

등 운영에 가까운 선택을 했습니다.

또한, 의도적으로 장애를 발생시켜

- 프로세스 종료
- 에러 발생
- OOM 상황

을 테스트하면서,  
systemd를 통한 자동 복구가 제대로 동작하는지 확인했습니다.

> 작은 기능이더라도  
> "안정적으로 돌아가려면" 생각보다 많은 작업이 필요하다는 것을 배웠습니다.

---

### 감성 분석 시스템 구축과 운영

자동매매 봇과 별도로,  
뉴스 텍스트 데이터를 기반으로 **감성 분석 모델**을 구축하여  
지금도 일정 주기로 실행하고 있습니다.

모델은 성능보다는  
"서버에서 안정적으로 돌아가는 경량 모델"을 목표로 구성했으며,

- 텍스트 정규화
- 토큰화
- 간단한 분류 모델 적용

등의 최소한의 처리로 감성 점수를 계산합니다.

각 실행 시 생성된 데이터는

- timestamp  
- 감성 점수  
- 텍스트 ID  

형태로 저장되며,  
시간이 지날수록 누적되는 구조입니다.

> “완성된 분석 모델”이 아니라,  
> “데이터를 쌓고, 나중에 의미를 찾기 위한 준비 과정”입니다.

---

### 자동화와 장애 대응

감성 분석 시스템은

- 크론탭
- 백그라운드 실행
- 재시도 로직

으로 관리하고 있으며,  

운영 중 다음과 같은 장애를 경험했습니다:

- API 오류
- 타임아웃
- 인코딩 문제

그때마다

- 예외 처리
- 로그 기록
- 재시도 전략

등을 적용하면서 개선했습니다.

> 모델을 만드는 것보다  
> "모델이 계속 돌아가게 만드는 것"이 더 어려웠습니다.

---

### 두 시스템의 공통된 학습 경험

두 프로젝트는 성격이 다르지만,  
학습 과정에서 다음과 같은 공통점을 발견했습니다:

1. 기능 구현보다 **운영 안정성**이 더 중요하다  
2. 리소스 제약은 **설계 방식**을 변화시킨다  
3. 장애는 피할 수 없으며, **기록과 복구**가 핵심이다  
4. 데이터는 한 번에 완성되지 않고, **시간에 따라 쌓인다**  
5. 작은 프로젝트라도 **실제 환경에서 운영**해보는 것이 가장 빠른 학습이다

---

### 현재 상태와 이후 계획

현재 두 시스템 모두

- 작은 규모로
- 안정적으로
- 자동화된 방식으로

운영되고 있습니다.

감성 데이터는 계속 쌓이고 있으며,  
향후 시장 데이터와 결합하여  
단순한 상관분석 또는 전략 테스트를 해볼 계획입니다.

자동매매 봇 역시  
더 구체적인 전략 또는 실제 데이터 연동으로 확장할 여지가 있습니다.

---

### 정리

결과물의 완성도는 높지 않지만,  
학습자 관점에서 매우 중요한 경험이었습니다.

- 서비스를 올려보고  
- 돌아가게 만들고  
- 문제가 생기면 고치고  
- 데이터를 쌓고  
- 환경을 옮기고  

이러한 반복이  
“이론을 알고 있다”와  
“직접 해봤다”의 차이라고 느꼈습니다.

> 작게 시작했지만,  
> 실제 환경을 경험하며 배우는 것이  
> 가장 유의미한 성장이라고 생각합니다.
